# Requirements Quality

## When to Use This Skill

- Reviewing requirements
- Validating specifications
- Improving requirement quality

## Core Concepts

### Quality Checklist

**Clarity:**
- ✅ Unambiguous language
- ✅ Single interpretation
- ✅ No technical jargon without definition

**Completeness:**
- ✅ All scenarios covered
- ✅ All data elements defined
- ✅ All error conditions addressed

**Consistency:**
- ✅ No conflicting requirements
- ✅ Consistent terminology
- ✅ Aligned with business goals

**Testability:**
- ✅ Measurable criteria
- ✅ Observable outcomes
- ✅ Clear acceptance criteria

**Feasibility:**
- ✅ Technically achievable
- ✅ Within budget/timeline

### Individual Requirement Quality

```markdown
FR-001: User Authentication
The system shall authenticate users via SSO when accessing the application.

- Priority: Must Have
- Source: Security Team
- Acceptance Criteria: User can login with SSO provider
```

## Validation Techniques

- Requirements review meetings
- Prototyping
- Use case validation
- Test case validation
